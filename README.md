# SQL Data Analysis Project (PostgreSQL)

## Overview
This repository contains structured SQL queries built using PostgreSQL to perform data analysis on a retail sales dataset.  
The project focuses on solving practical business problems using core and intermediate SQL concepts.

---

## Dataset Description

The dataset includes retail transactional data containing:

- Orders information  
- Returns data  
- Regional management details  

Primary tables used:

- `orders_clean`
- `returns_clean`
- `managers_clean`

---

## Project Structure

The repository is organized into concept-based SQL files:

- `01_basic_queries.sql`
- `02_filtering_and_sorting.sql`
- `03_aggregate_functions.sql`
- `04_group_by_having.sql`
- `05_joins.sql`
- `06_subqueries.sql`
- `07_case_statements.sql`
- `08_window_functions.sql`
- `09_business_problems.sql`
- `10_date_functions.sql`


---

## SQL Concepts Covered

- Data filtering using WHERE
- Sorting using ORDER BY
- Aggregate functions (SUM, AVG, COUNT, MAX, MIN)
- GROUP BY and HAVING
- INNER JOIN and LEFT JOIN
- Subqueries (scalar, correlated, derived tables)
- CASE statements for classification
- Window functions (ROW_NUMBER, RANK, DENSE_RANK, LAG, Running Total)
- Time-based analysis using date functions

---

## Business Problems Implemented

Examples of applied analysis include:

- Net profit after returns by region
- Top 3 customers by sales in each region
- Customer return rate percentage
- Region contribution to total company sales
- Month-over-month sales growth
- Yearly and monthly performance analysis

---

## Tools Used

- PostgreSQL
- Structured Query Language (SQL)

## Objective

The objective of this project is to demonstrate practical SQL skills required for:

- Data Analyst roles
- Business Intelligence roles
- Internship positions

The focus is on structured query writing, logical thinking, and solving real-world business problems using SQL.
